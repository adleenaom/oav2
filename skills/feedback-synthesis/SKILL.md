# Skill: Qualitative Feedback Synthesis

## When to Use
When qualitative feedback (user interviews, beta responses, usability notes) is uploaded and needs to be summarized into actionable product insights.

## When NOT to Use
- For quantitative/analytics data — use Looker Studio for that
- When feedback is too sparse to draw patterns (flag this)

## Inputs
- Uploaded feedback document or text (interview notes, survey responses, usability observations)
- Optional: current feature context or the specific area being tested

## Ideal Output
- A structured summary with:
  - **Top patterns** (what users repeatedly said or did)
  - **Pain points** (friction, confusion, drop-off moments)
  - **Positive signals** (what's working)
  - **Suggested actions** (prioritized by impact on completion/retention)
- Written in plain language — no AI-sounding summaries

## Process
1. Read all feedback before drawing conclusions
2. Group by theme, not by user
3. Prioritize patterns that affect completion rate and retention (core business goal)
4. Suggest 2–3 concrete actions ranked by effort vs. impact
5. Flag if feedback sample is too small to generalize

## Common Failure Modes
- **Over-indexing on outliers:** Note when a finding comes from only 1–2 users
- **Vague summaries:** Every insight must connect to a specific user behavior or quote
- **Missing action:** Synthesis without suggested next steps is incomplete

## Success Criteria
1. Patterns are grounded in specific feedback, not generalizations
2. Suggested actions directly address completion rate or retention
3. Summary is scannable and written in approachable, non-corporate language
