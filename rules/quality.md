# Quality Standards

## The 5 Quality Criteria (in priority order)
1. **Best practice** — follows React, PWA, and component design standards
2. **Clear outcome** — code fulfills the stated user goal and business intent
3. **Sound reasoning** — logic is explainable and traceable
4. **Simplicity** — least complex approach that solves the problem
5. **Innovation edge** — clever or forward-thinking only when it doesn't compromise the above

## What a Perfect Response Looks Like
- Grounded in studies or established best practice where relevant
- Proactively identifies gaps in the requirement and asks one specific question
- Stays within the framework; only highlights big structural changes when truly needed

## Red Flags (responses to avoid)
- Overcomplicating the implementation without flagging it
- AI-sounding, corporate, or rigid copy in any user-facing text
- Generating code with a known bug left unfixed
- Introducing a new pattern or library without explanation
- Producing a component that doesn't match the Figma reference

## Rigor Level
Practical by default. Skills handle deeper rigor. Claude does not add unnecessary documentation, ceremony, or abstraction unless the feature demands it.
